# Recursion in Programming

## What is Recursion?
Recursion is a technique where a function calls itself to solve a problem by breaking it down into smaller subproblems.

## Key Components

### 1. Base Case
The condition that stops the recursion.
```
if n == 0:
    return 1  # Base case
```

### 2. Recursive Case
The function calls itself with a simpler input.
```
return n * factorial(n - 1)  # Recursive case
```

## Common Examples

### Factorial
```
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

### Fibonacci
```
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

## Advantages
- Elegant and readable for certain problems
- Natural for tree/graph traversal
- Mirrors the problem definition

## Disadvantages
- Can be inefficient (repeated calculations)
- Uses more memory (call stack)
- Risk of stack overflow for large inputs

## Optimization Techniques
- **Memoization**: Cache results to avoid recalculation
- **Tail Recursion**: Optimize for tail-recursive calls
- **Iteration**: Converting recursion to loops
